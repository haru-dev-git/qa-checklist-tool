# QAチェックリスト生成ツール

## 概要

QAチェックリスト生成ツールは、Webアプリや業務機能の確認項目を、正常系・異常系・境界値などの観点で整理するためのブラウザ完結型Webアプリです。

プロジェクト名、機能名、機能概要、カテゴリ、チェック項目を入力し、チェックリストとして管理できます。  
作成したチェック項目は、実施済み/未実施の切り替え、削除、localStorage保存、CSV出力に対応しています。

サーバー、データベース、外部APIは使用せず、HTML/CSS/JavaScriptのみで実装しています。

## 使い方

1. プロジェクト名を入力する
2. 機能名を入力する
3. 必要に応じて機能概要を入力する
4. カテゴリを選択する
5. チェック項目を入力する
6. 「チェック項目を追加」を押す
7. 必要に応じて実施済み/未実施を切り替える
8. 保存する場合は「保存する」を押す
9. CSVとして出力する場合は「CSV出力」を押す
10. データを削除する場合は「全クリア」を押す

## 入力値チェック

以下の入力値チェックを実装しています。

| 項目 | 条件 |
|---|---|
| プロジェクト名 | 必須、50文字以内 |
| 機能名 | 必須、50文字以内 |
| 機能概要 | 任意、300文字以内 |
| チェック項目 | 必須、100文字以内 |

未入力や文字数超過がある場合は、画面上にエラーメッセージを表示します。

## データ保存

入力内容とチェック項目は、ブラウザのlocalStorageに保存できます。  
保存後にページを更新しても、保存済みのプロジェクト名、機能名、機能概要、チェック項目、実施状態が復元されます。

保存対象は以下です。

- プロジェクト名
- 機能名
- 機能概要
- チェック項目一覧
- カテゴリ
- 実施状態
- 作成日時
- 次に使用するID

## CSV出力

作成したチェックリストはCSV形式で出力できます。

CSVには以下の項目を出力します。

- プロジェクト名
- 機能名
- 機能概要
- ID
- カテゴリ
- チェック項目
- 実施状態
- 作成日時

日本語の文字化けを防ぐためにBOMを付与し、カンマや改行を含む文字列でも列が崩れにくいように各値をダブルクォーテーションで囲んでいます。

## テスト観点

以下の観点で動作確認を行いました。

### 正常系

- 正常に1件追加できる
- 複数件追加できる
- チェック状態を切り替えられる
- 削除できる
- 保存できる
- ページ更新後に復元できる
- CSV出力できる
- 全クリアできる

### 異常系

- プロジェクト名未入力時にエラーが表示される
- 機能名未入力時にエラーが表示される
- チェック項目未入力時にエラーが表示される
- チェック項目0件でCSV出力するとエラーが表示される

### 境界値

- プロジェクト名50文字で追加できる
- プロジェクト名51文字でエラーになる
- チェック項目100文字で追加できる
- チェック項目101文字でエラーになる

## 開発背景

IT領域でQA、テスト、仕様整理、SaaS導入支援、Web運用改善などに関心があり、仕様を整理して確認観点に落とし込む力を示すために作成しました。

単に画面を作るだけではなく、正常系・異常系・境界値などのテスト観点を意識し、入力値チェック、保存、CSV出力まで実装することで、小さな業務支援ツールとして使える形を目指しました。

## 工夫した点

### 1. QA観点をカテゴリとして整理

チェック項目を正常系、異常系、境界値、表示確認、操作確認、データ保存などに分類できるようにしました。

自由入力ではなくカテゴリを固定選択にすることで、表記揺れを防ぎ、チェックリストとして整理しやすい形にしています。

### 2. 入力値チェックを実装

未入力や文字数超過時にエラーメッセージを表示するようにしました。

特に、プロジェクト名、機能名、チェック項目は必須入力とし、チェックリストとして最低限必要な情報が欠けないようにしています。

### 3. localStorageによる保存・復元

入力内容とチェック項目をlocalStorageに保存し、ページ更新後も復元できるようにしました。

JavaScriptのオブジェクトをJSON文字列に変換して保存し、読み込み時にJSON.parseで元のデータ構造に戻しています。

### 4. CSV出力に対応

作成したチェックリストをCSV形式で出力できるようにしました。

CSV出力では、実施状態や作成日時も含めることで、確認結果を外部で共有・記録しやすい形にしています。

### 5. 小さく作って段階的に改善

最初から多機能にせず、MVPとして以下の順番で実装しました。

1. HTMLで画面の骨組みを作成
2. CSSで最低限の見た目を調整
3. チェック項目追加機能を実装
4. チェック状態切り替えと削除を実装
5. localStorage保存・復元を実装
6. CSV出力を実装
7. テスト観点を整理
