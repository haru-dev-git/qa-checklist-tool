# QAチェックリスト生成ツール 設計書

## 1. アプリ概要

QAチェックリスト生成ツールは、Webアプリや業務機能の確認項目を、正常系・異常系・境界値などの観点で整理するためのブラウザ完結型Webアプリです。

ユーザーはプロジェクト名、機能名、機能概要、カテゴリ、チェック項目を入力し、チェックリストとして管理できます。

## 2. 開発目的

本アプリは、単なるチェックリスト作成ツールではなく、以下の力を示すために作成しました。

- 仕様を整理する力
- 正常系、異常系、境界値などのテスト観点を考える力
- 入力値チェックを実装する力
- JavaScriptでDOM操作を行う力
- localStorageでデータを保存・復元する力
- CSV形式でデータを出力する力

## 3. 対象ユーザー

- 個人開発者
- QA初学者
- Web制作やWebアプリの確認担当者
- SaaS導入や運用確認を行う担当者

## 4. 使用技術

## 8. 入力項目

| 項目 | 入力形式 | 必須 | 文字数制限 |
|---|---|---:|---:|

| 機能名 | テキスト | 必須 | 50文字以内 |

| カテゴリ | セレクトボックス | 必須 | 固定選択 |

## 9. カテゴリ

- 正常系
- 異常系

- 表示確認
- 操作確認
- データ保存
- その他

## 10. データ設計

チェック項目はJavaScriptの配列で管理します。

```javascript
let checkItems = [
  {
    id: 1,
    category: "正常系",
    text: "正しいメールアドレスとパスワードでログインできる",
    checked: false,
    createdAt: "2026-06-08T00:00:00.000Z"
  }
];
````

### 各プロパティの役割

| プロパティ     | 内容               |
| --------- | ---------------- |
| id        | チェック項目を一意に識別するID |
| category  | 正常系、異常系などのカテゴリ   |
| text      | チェック項目本文         |
| checked   | 実施済みかどうか         |
| createdAt | 作成日時             |

## 11. localStorage設計

localStorageには、以下の形式で保存します。

```javascript
{
  projectName: "ECサイト確認",
  featureName: "ログイン機能",
  featureDescription: "メールアドレスとパスワードでログインする機能",
  checkItems: [
    {
      id: 1,
      category: "正常系",
      text: "正しい情報でログインできる",
      checked: false,
      createdAt: "2026-06-08T00:00:00.000Z"
    }
  ],
  nextId: 2
}
```

保存キー名：

```text
qaChecklistData
```

## 12. CSV出力設計

CSVには以下の項目を出力します。

| 列名      | 内容           |
| ------- | ------------ |
| プロジェクト名 | 入力されたプロジェクト名 |
| 機能名     | 入力された機能名     |
| 機能概要    | 入力された機能概要    |
| ID      | チェック項目ID     |
| カテゴリ    | チェック項目カテゴリ   |
| チェック項目  | チェック項目本文     |
| 実施状態    | 実施済み/未実施     |
| 作成日時    | チェック項目の作成日時  |

CSV出力時は、日本語の文字化けを防ぐためBOMを付与します。  
また、カンマや改行を含む文字列で列が崩れにくいよう、各値をダブルクォーテーションで囲みます。

## 13. 処理設計

### チェック項目追加

1. 入力欄の値を取得する
2. 未入力・文字数超過を確認する
3. 問題がなければチェック項目オブジェクトを作成する
4. `checkItems` 配列に追加する
5. 入力欄をクリアする
6. チェックリスト表示を更新する
7. 件数表示を更新する

### チェック状態切り替え

1. チェックボックスの `data-id` から対象IDを取得する
2. `checkItems` 配列から対象項目を探す
3. `checked` を `true / false` に切り替える
4. 画面を再描画する
5. 件数表示を更新する

### 削除

1. 削除ボタンの `data-id` から対象IDを取得する
2. 対象ID以外の項目だけを残す
3. 画面を再描画する
4. 件数表示を更新する

### localStorage保存

1. プロジェクト名、機能名、機能概要を取得する
2. チェック項目一覧と次のIDをまとめる
3. `JSON.stringify()` で文字列化する
4. localStorageに保存する

### localStorage読み込み

1. localStorageから保存データを取得する
2. 保存データがなければ何もしない
3. `JSON.parse()` でオブジェクトに戻す
4. 入力欄とチェック項目一覧に反映する
5. 画面を再描画する

### CSV出力

1. チェック項目が存在するか確認する
2. CSVのヘッダー行を作成する
3. チェック項目をCSV用の行に変換する
4. CSV文字列を作成する
5. Blobを作成する
6. ダウンロードリンクを一時的に作成する
7. CSVファイルとしてダウンロードする

## 14. 入力値チェック仕様

| 条件             | 表示メッセージ                  |
| -------------- | ------------------------ |
| プロジェクト名未入力     | プロジェクト名を入力してください。        |
| 機能名未入力         | 機能名を入力してください。            |
| チェック項目未入力      | チェック項目を入力してください。         |
| プロジェクト名51文字以上  | プロジェクト名は50文字以内で入力してください。 |
| 機能名51文字以上      | 機能名は50文字以内で入力してください。     |
| 機能概要301文字以上    | 機能概要は300文字以内で入力してください。   |
| チェック項目101文字以上  | チェック項目は100文字以内で入力してください。 |
| チェック項目0件でCSV出力 | CSV出力するチェック項目がありません。     |

## 15. テスト観点

### 正常系

* 正常に1件追加できる
* 複数件追加できる
* チェック状態を切り替えられる
* 削除できる
* 保存できる
* ページ更新後に復元できる
* CSV出力できる
* 全クリアできる

### 異常系

* プロジェクト名未入力時にエラーが表示される
* 機能名未入力時にエラーが表示される
* チェック項目未入力時にエラーが表示される
* チェック項目0件でCSV出力するとエラーが表示される

### 境界値

* プロジェクト名50文字で追加できる
* プロジェクト名51文字でエラーになる
* チェック項目100文字で追加できる
* チェック項目101文字でエラーになる

## 16. 手戻りしやすい箇所

* 追加ボタンのイベント登録が重複すると、1回のクリックで複数件追加される
* チェック状態切り替え後に件数表示を更新し忘れる可能性がある
* 削除後に再描画しないと画面とデータがずれる
* 保存後にチェック状態を変更しても、再保存しないとlocalStorageには反映されない
* CSV出力時にカンマや改行を考慮しないと列が崩れる
* 成功メッセージとエラーメッセージの色が混在する可能性がある
